# ChatGPT / GPT Image 官方提示词规则总结

来源：OpenAI Cookbook《GPT Image Generation Models Prompting Guide》，发布于 2026-04-21。

用途：以后用户说“用 ChatGPT 绘图”“用 GPT 绘图”“用 ChatGPT image2 / gpt-image 生成或编辑图片”时，优先参考本规则来组织提示词方案；如果任务同时涉及参考图改风格与固定正交斜俯视游戏视角，还要叠加 `rules/reference-style-orthographic-game-view.md`。

## 一、模型与参数选择

- 新建或高质量工作流默认优先考虑 `gpt-image-2`。
- `gpt-image-2` 适合高质量生成、图片编辑、文字较多的图片、写实、合成、身份敏感编辑，以及希望减少返工次数的生产流程。
- 如果速度、成本或大批量试验更重要，可以先用 `quality="low"` 验证；如果是小字、密集信息图、近景人像、身份保持或高分辨率输出，应比较 `medium` / `high`。
- 旧工作流迁移时，不要一开始大改 Prompt；先保留原 Prompt 对比质量、延迟和返工率，再做微调。

## 二、通用提示词结构

OpenAI 官方建议稳定使用清晰、可维护的结构，而不是追求复杂语法。推荐顺序：

```text
背景 / 场景 → 主体 → 关键细节 → 构图 / 镜头 / 光影 → 需要改变的内容 → 必须保持不变的内容 → 负面约束 → 输出用途 / 质量要求
```

复杂需求不要写成一大段，应该使用短标题、分段或列表，让目标、素材、约束更容易扫描。

## 三、描述要具体

- 明确视觉媒介：照片、3D 渲染、2D 手绘、像素艺术、水彩、UI mockup、信息图等。
- 明确材质、形状、纹理和细节：玻璃、金属、毛绒、纸张纤维、磨损边缘、真实皮肤纹理等。
- 写实任务中，直接写 `photorealistic` 有助于进入写实模式；也可以使用真实相机、专业摄影、自然光等高层级摄影语言。
- 相机、镜头和物理参数不要当成严格模拟器，应主要用于控制整体观感、构图和氛围。

## 四、构图、视角和布局

提示词应主动指定：

- 景别：特写、半身、全身、宽景、地图视角等。
- 视角：平视、低角度、俯视、正交斜俯视、top-down 等。
- 布局：主体居中、左侧留白、logo 右上角、文字放底部、四宫格分镜等。
- 光影和情绪：柔光、黄金时刻、低调冷光、霓虹、高对比、清晨雾气等。

如果画面需要很强的情绪，如夜景、雨景、霓虹、电影感或低光环境，要额外描述空间尺度、空气感和色彩，否则模型可能只给表层风格。

## 五、人物、姿势和动作

人物类 Prompt 要明确：

- 人物在画面中的比例和景别。
- 身体是否完整可见，例如全身、脚部入镜、手部清楚。
- 视线方向，例如看向书本、看向远处、不要看镜头。
- 与物体的互动方式，例如双手自然握住把手、正在整理渔网、跪在地上帮助动物。

这些描述有助于控制身体比例、动作几何关系和视线方向。

## 六、编辑任务：先说改什么，再锁定不变项

用于 image-to-image、局部修改、去 UI、改风格、换天气、换材质等任务时，必须明确拆分：

```text
Change only: 【只改变的内容】
Preserve exactly: 【必须保持的内容】
Do not change: 【禁止改变的内容】
```

官方反复强调：编辑任务要清楚区分“应该改变什么”和“必须保持什么”，并在每轮迭代中重新写出关键不变项，减少漂移。

### 编辑任务常用不变项

- 原图构图
- 镜头角度
- 透视关系
- 空间布局
- 主体身份 / 脸 / 体型 / 发型 / 姿势
- 背景
- 光照方向
- 阴影
- 周围物体
- 品牌元素
- 标签文字
- 道路、建筑、水面、树木等地图位置关系

## 七、参考图风格迁移

风格迁移要同时说明：

1. 从参考图继承什么风格线索：色彩、纹理、笔触、材质、光影、颗粒感、线条语言。
2. 新图要生成或改变什么主体 / 场景。
3. 哪些内容不能新增或漂移：背景、构图、视角、额外元素、文字、logo、水印等。

推荐句式：

```text
Use the uploaded image as the reference. Preserve 【layout / camera / structure】. Change only 【style / material / mood】. Keep 【objects / positions / proportions】 unchanged. Do not add extra elements, text, logos, or watermarks.
```

## 八、多图输入与合成

多张参考图时，不要只说“参考这些图”，要给每张图编号并说明用途：

```text
Image 1: source scene / layout reference.
Image 2: style reference.
Image 3: character reference.
Apply Image 2's style to Image 1 while preserving Image 1's layout. Insert the character from Image 3 at 【位置】, matching lighting, scale, perspective, and shadows.
```

合成任务必须说明对象来自哪张图、放在哪里、保留什么不变，以及如何匹配光照、比例、透视和阴影。

## 九、文字出现在图片里时

如果图片中需要出现文字：

- 把要出现的文字放在引号里，要求逐字一致。
- 说明文字只能出现一次，不能多字、少字或额外添加。
- 指定字体风格、大小、颜色、位置、对比度和排版层级。
- 对小字、密集文字、图表、标签、说明面板，优先使用 `quality="high"`。
- 如果文字不重要或不希望出现，明确写：`No text, no labels, no watermark, no logo`。
- 不要默认输出 SD 式单独 Negative Prompt；ChatGPT / GPT 更适合把约束自然地写进完整指令，例如 `Do not change the pose, proportions, canvas ratio, or lighting style.`

## 十、不同任务的提示词写法

### 1. 信息图 / 教育图

像写教学设计 brief：说明受众、学习目标、视觉格式、必须包含的标签、箭头、步骤和科学约束。要求清晰图标、可读标签、足够留白，不要装饰过度。

### 2. 照片写实

像描述真实拍摄瞬间：自然光、真实材质、皮肤纹理、磨损、生活化细节、非摆拍。避免过度影棚感、过度修图、电影海报感。

### 3. Logo

说明品牌性格、用途、简洁原创标志、强轮廓、负空间、可缩放性、纯背景、无水印。避免复杂细节和侵权元素。

### 4. 广告图

像创意 brief：品牌、受众、文化语境、概念、构图、广告文案、视觉调性。若有广告语，必须逐字写清楚并限制额外文字。

### 5. 漫画 / 分镜

按 panel 拆分，每格一个清晰视觉动作。描述具体动作、人物位置、情绪变化和节奏，不要笼统讲故事。

### 6. UI mockup

像描述真实已上线产品：布局、层级、间距、真实界面元素、可用性、清晰字体。避免概念艺术语言。

### 7. 草图转成品图

保留草图意图、布局、比例和透视；增加合理材质、光照和环境；明确不要新增元素或文字。

### 8. 产品图 / 背景移除

强调边缘干净、轮廓清晰、无白边 / 光晕，保留产品几何和标签清晰度；如果需要透明图，可先输出不透明背景，再做后处理抠图。

### 9. 天气 / 时间 / 光照转换

只改变环境条件：光照方向、光质、阴影、氛围、降水、地面湿度等；保持身份、几何结构、镜头角度和物体位置不变。

### 10. 角色一致性 / 儿童书多页

先建立角色锚点：外观、比例、服装、表情、性格、配色；后续每张图重复这些不变项，只改变场景、动作和叙事。

## 十一、迭代方法

- 不要一开始把所有需求塞进超长 Prompt；先写干净基础版，再做小步迭代。
- 每次只改一个变量，如“光更暖”“去掉多余树木”“恢复原背景”。
- 关键不变项每次都要重复，不要只说“和上次一样”。
- 记录每次测试结果：哪里漂移、哪里成功、下一版要强化什么。

### 当前项目测试结论

- 2026-06-17 复测修正：此前 GPT 噪点问题主要来自绘图设置，不是提示词本身的问题。
- nanobanana PRO 和 ChatGPT 使用同一去噪优化提示词时效果接近，风格都更接近原参考图。
- 因此不要过度归因于平台差异；后续先检查模型、质量、重绘强度、参考图权重和输出设置，再判断提示词问题。
- 如果仍出现噪点，再补充：`低噪点、无颗粒感、无脏污纹理、干净色块、平滑渐变、不要随机纹理、不要过度锐化、不要杂色斑点`。
- 2026-06-22 角色三视图换装复测：GPT 自动分辨率可能输出 1:1 方图并造成三视图人物比例变形；角色三视图任务必须显式写明参考图原始横向比例、推荐 16:9 / 2048×1152 或同等横向尺寸，并禁止 1:1 方图、裁切、拉伸和压缩人物。
- 同一角色三视图换装任务中，中文提示词比英文提示词更稳定；后续优先用中文完整提示词。

## 十二、ChatGPT / GPT 绘图通用模板

```text
Task / 任务：生成或编辑【目标图片类型】，用于【用途】。

Reference / 参考图：
- 使用上传图片作为【布局 / 风格 / 角色 / 产品 / 场景】参考。
- 必须保留：【列出不变项：构图、视角、主体位置、身份、道具、文字等】。
- 只改变：【列出改变项：风格、材质、色彩、天气、时间、某个物体等】。

Scene / 场景：
【背景、空间、主体、道具、环境关系】

Style / 风格：
【视觉媒介、艺术风格、材质、纹理、色彩、光影、氛围】

Composition / 构图：
【画幅、景别、镜头角度、主体位置、留白、层次】

Constraints / 约束：
- Do not change: 【禁止改变项】
- No extra text, no watermark, no logo unless explicitly requested.
- Keep the image clean, readable, and production-ready.

Quality / 参数建议：
- `gpt-image-2`
- 草稿 / 批量测试：`quality="low"`
- 常规成图：`quality="medium"`
- 小字、密集图表、身份保持、高精度编辑：`quality="high"`
```
